# Changelog

All notable changes to the Spotykach firmware are recorded here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). This project does not yet expose a version constant in the firmware; entries are grouped by the Git tag / release they ship in.

## [Unreleased]

### Added

- **Buffer-based bipolar/reverse varispeed tape engine (`shuttle`).** `make ENGINE=shuttle` builds the random-access counterpart to the streaming `tape` engine: audio lives in SDRAM (not SD-streamed), so the read pointer is random-access and **reverse**, **freeze**, and **loop windowing** are trivial - the trade is a finite, RAM-capped length (currently 30 s/track). It runs **four independent mono tape tracks** - two per deck (A/B) - which free-run at their own speeds and sum into the stereo bus. Each track is a contiguous `float32` buffer carved from the SDRAM arena (1.44 M frames / 5.76 MB at 30 s/48 kHz; 4 tracks = 23 MB of the 48 MB arena), played by a **signed fractional read pointer** (`double`); reverse is a negative speed, freeze is speed 0 -> silence, ends wrap. `CapOwnDisplay | CapDualDeck | CapAux | CapAltPos | CapPitchPickup`. Host-verified (10 test groups), builds + links on target (~82% `SRAM_EXEC`), pending hardware bring-up. (`src/engine/shuttle/shuttle_engine.{h,cpp}`, `src/engine/engine_select.h`, `Makefile`, `docs/engines/shuttle.md`, `docs/dev/shuttle_impl.md`)

- **Shuttle PITCH = bipolar capstan speed; Play snaps to unity.** PITCH is a per-track bipolar speed knob: a deadzone about **noon = stop (silence)**, clockwise = forward to **+2x**, counter-clockwise = **reverse to -2x**. Unity (+1x) lands off-centre, so the **Play pad snaps** the focused track to +1x and arms a one-shot pickup re-seed so the absolute pot must be swept across unity before it retakes the speed. (`src/engine/shuttle/shuttle_engine.cpp`)

- **Shuttle four-track focus (Rev swap) + global re-align (Seq pad).** Each deck holds two tracks; all four play at once, but a deck's knobs (PITCH/POS/SIZE/MIX/Alt+PITCH) and ring address only the **focused** track. Tapping a deck's **Rev pad** swaps the focus (edrums mechanism: the backgrounded track keeps playing; a one-shot re-seed catches the pots without jumps). The four tracks drift out of phase by design; the **Seq pad re-aligns all four** to their loop start in one atomic gesture, declicked per audibly-rolling track (a ~1.3 ms fade-out -> jump -> fade-in, applied in the ISR off a flag so it is race-free and all four snap on the same block). (`src/engine/shuttle/shuttle_engine.{h,cpp}`)

- **Shuttle loop window (POS / SIZE).** Each track plays a window `[start, start+length)` of its buffer rather than the whole recording: **SIZE** sets the length (full buffer down to a `kMinLoopFrames` stutter floor), **POS** slides the start across the uncovered tail (the window always stays inside the recording; at SIZE = full, POS is inert). The varispeed pointer wraps within the window both directions, and the Seq re-align snaps to the window start. Both are per-track (repoint on a Rev swap). (`src/engine/shuttle/shuttle_engine.cpp`)

- **Shuttle audio source: record + SD load.** A track fills its buffer by **recording** the live input (Alt+Play, overwrite from the start) or by **loading** an existing `/shuttle/` WAV from the card into RAM (Alt+PITCH selects the slot), drained over a few `prepare()` passes; the drain advances by `play_consume`'s actual returned byte count (zero-fill is destination-only), so an underrun is re-pulled, never baked in as silence. The shuttle keeps its files in its **own `/shuttle/` folder** (filenames `tape_<a|b>_<n>.wav`), isolated from the streaming `tape` engine's `/tapes/` so the two engines never share or overwrite each other's takes. (`src/engine/shuttle/shuttle_engine.cpp`, `src/hw/fat_file.cpp`)

- **Shuttle boot preload (all four tracks).** On power-up the shuttle fills **all four tracks** from the card if their files exist - track *t* of each deck loads slot *t*'s file (deck A: `tape_a_1/_2.wav` -> tracks 0/1; deck B: `tape_b_1/_2.wav`) - so a freshly powered unit is ready to jam without a manual Alt+PITCH load (the RAM engine can't stream lazily the way `tape` does). Per-track conditional (an absent slot file leaves that track empty) and, since a deck's two tracks share one SD stream, **serialized per deck** (the two decks load in parallel). The card mounts cooperatively early in boot, so the load waits across `prepare()` passes until the volume is up, then is abandoned after a boot deadline if there is no card / no files. (`src/engine/shuttle/shuttle_engine.{h,cpp}`)

- **Shuttle per-track pan + routing switch.** The four tracks now pan **individually** into the stereo bus - previously each deck summed its two tracks to mono before a single deck-level pan, so the two tracks of a deck could only share one pan position. The panel **routing switch** (`ConfigId::Route`, shown on the mode L/C/R LED) picks where each track's pan comes from: **LEFT** (DoubleMono) = the manual per-track Alt+POS pan; **CENTRE** (Stereo) = auto-spread, a deck's two tracks fanned hard L/R for instant stereo width; **RIGHT** (GenerativeStereo) = a random equal-power pan per track (LCG, no platform RNG), rolled **once on entry** - unlike the tape engine, shuttle guards the roll on the actual switch transition because the platform calls `set_config` every loop. Alt+POS became a per-track knob (was per-deck), so the platform now also re-seeds `ParamId::AltPos` on a Rev focus swap to keep the pot pickup from jumping. (`src/engine/shuttle/shuttle_engine.{h,cpp}`, `src/ui/core.ui.cpp`)

- **Shuttle idle ring reads as "on", with a per-track hue.** A loaded-but-stopped deck previously painted its ring black (looked powered-off); it now breathes a dim standby glow (a slow raised-cosine pulse off the engine's own clock) in the **focused track's hue** - track 1 indigo, track 2 amber - so a glance tells you which of a deck's two tracks is in focus. Live transport keeps its direction colors (record red / forward green / reverse cyan / frozen dim-white), and the Rev-swap flash now uses the newly-focused track's hue so the color change itself signals the swap. (`src/engine/shuttle/shuttle_engine.cpp`)

- **`CapPitchPickup`: route PITCH through the pickup-gated path.** A new capability so an engine that snaps the speed (shuttle's Play->unity) can hold the snap via `take_param_reseed` - the absolute PITCH pot must be swept across the snapped value before it retakes control. Engines without it keep raw (un-gated) PITCH byte-for-byte. (`src/engine/engine_params.h`, `src/ui/core.ui.{h,cpp}`)

- **Headless shuttle test.** `make -C host test-shuttle` runs `host/test_shuttle.cpp` (10 groups, via a controllable `FakeClock` for the pad debounce and a per-deck `FakeStream` for the SD-load drain): bipolar speed map, record + unity playback, reverse-wrap + silence-at-noon + loop, Play->unity snap (one re-seed), Rev swap + track independence, SD load, Seq re-align (all four atomic), realign declick (no large output step across the jump), the POS/SIZE loop window, and the boot preload (fills all four tracks from their slot files, serialized per deck, skipping a track whose file is absent). (`host/test_shuttle.cpp`, `host/Makefile`)

- **Audio-prep script + docs for the streaming format.** Both streaming engines load only mono 32-bit-float WAV at 48 kHz, so `scripts/convert_tape_audio.py` batch-converts arbitrary sources (via ffmpeg or sox), verifies each output header against the firmware's requirement, and can name files straight into deck slots (`--engine tape|shuttle`, `--deck a|b`, `--start-slot`); it warns past the shuttle's ~30 s RAM cap. `docs/preparing-audio.md` documents the format and the ffmpeg/sox one-liners, linked from the tape and shuttle engine docs. (`scripts/convert_tape_audio.py`, `docs/preparing-audio.md`, `docs/engines/{tape,shuttle}.md`)

- **Tape: post-FX resonant low-pass filter on the grit pad, soft-limited bus.** Each tape deck gains a 2-pole resonant low-pass after the wow/flutter + Jiles-Atherton chain - a dub/tape tone control. It rides the **grit knob-edit gesture** rather than dedicated knobs: hold **grit + PITCH** = cutoff (exp, ~40 Hz .. 20 kHz), hold **grit + MIX** = resonance (Q 0.7 .. ~10, squared so the low half of the knob stays gentle), reusing the platform's existing `GritIntensity`/`GritMix` params - the tape engine just reinterprets them, so there is **no CoreUI/platform change**. Cutoff boots **open** so the filter is inert until swept down (its pickup is seeded open -> turn down to engage). The summed two-deck output bus is then **soft-limited** (`daisysp::SoftLimit`, the cubic soft-clip the edrums/granular buses use) so the resonant peak and two simultaneous decks can't clip the codec. Host-verified (low cutoff attenuates a 6 kHz tone, resonance boosts near the corner, limiter caps the peak at ~0 dBFS); pending hardware bring-up. (`src/engine/tape/tapefx.dsp`, `faust_kernel_tapefx.h`, `tape_engine.{h,cpp}`, `host/test_tape.cpp`, `docs/engines/tape.md`)

- **CMake build at parity with the Makefile.** The opt-in CMake path now resolves `reverb`, `shuttle`, and `gigaverb` (the gen~ engine's full define/include/warning set), and the `Makefile.cmake` frontend gains the matching `engine-*` flash targets plus the host-side `faust-gen`/`gen-engines`/`test-scripts` targets. All three configure and compile; `gigaverb` linked end-to-end. The Make build stays canonical. (`Makefile.cmake`, `CMakeLists.txt`)

### Changed

- **Tape wow/flutter rate favors very low frequencies.** The MODFREQ -> rate map was linear with too high a floor (`0.5 .. 2.5 Hz` wow / `6 .. 12 Hz` flutter), so even the lowest setting felt too fast. It is now a **cubic** curve with a low floor (`0.1 .. 2.5 Hz` wow / `0.5 .. 12 Hz` flutter): the bottom of the knob is a slow drift (~10 s period) and the rate climbs to a reasonable wobble only near the top. Endpoints (top speed) are unchanged. (`src/engine/tape/tapefx.dsp`, `faust_kernel_tapefx.h`)

- **gen~ engine build targets drop the `gen_` prefix.** The gigaverb engine renames `gen_gigaverb` -> `gigaverb` (`ENGINE=gigaverb`, `src/engine/gigaverb/`, `SPK_ENGINE_GIGAVERB`), and the generator convention changes to match: `scripts/gen_engine.py` now wires a gen~ export as a plain `ENGINE=<name>` / `src/engine/<name>/` / `SPK_ENGINE_<NAME>` (no `gen_`/`SPK_ENGINE_GEN_`), so a gen~ engine sits beside native and Faust engines with no prefix - and the only `gen`-named directory is the shared infra `src/engine/gen/`. The gen~ provenance now lives in the engine's docs + `manifest.json`, not its target name. (`scripts/gen_engine.py`, `Makefile`, `src/engine/engine_select.h`, `src/engine/gigaverb/`, `docs/engines/gigaverb.md`, `docs/engine-types/gen.md`, `docs/engine-layout.md`, `README.md`)

- **SD streaming service gated by capability (`SPK_USE_STREAM`), not engine name.** The streaming platform (`StreamDeck`/`FatFile`, SDRAM rings, keep-card-mounted) was guarded by `#if defined(SPK_ENGINE_TAPE)`; it now keys on a Makefile-set `SPK_USE_STREAM` flag that both `tape` and `shuttle` define. The `app.cpp` construct/pump/inject of `_stream` is the one shared touch point (a hardware service the platform must own); the rest now reference the capability, so the next streaming engine is a one-line Makefile change. Non-streaming engines stay byte-identical (`tape` output unchanged - both macros are defined for a tape build). (`Makefile`, `src/app.cpp`, `src/hw/buffer.sdram.{h,cpp}`, `src/hw/stream_deck.cpp`, `src/hw/fat_file.cpp`, `src/memory/storage.h`)

### Fixed

- **Streaming WAV load: validate format + tolerate metadata chunks (was silent distortion / no-load).** The shared `WavStreamReader` (tape + shuttle playback/load) handed a file's raw body bytes straight into the engine's mono-float frames with no format check, so a 16-bit, 32-bit-**integer**, or stereo WAV was reinterpreted bit-for-bit as float32 - extreme full-scale distortion (a small int sample becomes a ~1e13 float). It also parsed only the first 64 header bytes, so an externally-authored file whose `data` chunk sits behind a `fact`/`LIST` metadata block (e.g. ffmpeg/DAW float output lands it at offset 90-128) silently failed to load. `begin()` now (a) **walks the RIFF chunk list by seeking** - skipping any `fact`/`LIST`/`JUNK` metadata of any size with no fixed offset ceiling, and handling `WAVE_FORMAT_EXTENSIBLE` - to locate `data`, exactly as a conformant reader must, and (b) rejects anything that is not the native record format - `AudioFormat`/`BitsPerSample` per the build (float32 default, int16 under `LOFI_INT16`), mono, 48 kHz - returning false so `start_play` raises the deck's error flash instead of playing garbage. The format checks are engine-capability gates (the stream path does no conversion), not WAV-spec strictness. Device-recorded tapes (canonical mono-float 44-byte header) are unaffected. New host tests cover the rejects (16-bit / int / stereo / non-48k), the native accept, and parsing `data` behind `fact`/`LIST` metadata at any offset (incl. the real ffmpeg/Logic layout, a chunk before `fmt `, and odd-sized chunks). (`src/memory/wav_stream.h`, `host/test_stream.cpp`)

- **Tape FX boots OFF (was a colored, un-reducible default).** The per-deck wow/flutter + Jiles-Atherton FX seeded its four knobs at non-zero defaults (drive/char/wow 0.3, rate 0.4), so a freshly loaded tape played with audible wow/flutter and saturation. Worse, the platform seeds each knob's pickup from those values, so a pot physically below the seed could never be swept down across it (soft-takeover) - the wow/flutter literally could not be turned off (it read as a "record needle moving between grooves"). `_fx_n` now defaults to all-zero per deck: the FX boots neutral and every param is freely reducible from zero. (`src/engine/tape/tape_engine.h`)

- **Audio-prep script writes the canonical WAV header itself (was noise on playback).** `scripts/convert_tape_audio.py` let ffmpeg/sox author the WAV container, which for 32-bit float **always** carries a `fact` chunk (and a DAW source's `LIST`/INFO metadata rode along), pushing the `data` body to offset 58-128 instead of the canonical 44 - all spec-valid, but a firmware reader that keys on the canonical offset then played that metadata as audio (structured noise). The script now decodes to **raw float** (`ffmpeg -f f32le` / `sox -t f32`) and writes a byte-exact copy of the firmware's own 44-byte header (`data` at 44, no `fact`/`LIST`), so output plays on any reader regardless of how its header parser works. Pairs with the chunk-walking reader above: either fix alone resolves the noise. (`scripts/convert_tape_audio.py`)

- **Tape: a wrong-format file now flashes distinctly from an empty slot.** A rejected load **strobes** the deck's amber error indicator (~4.5 Hz) when the file exists but was refused for its format, versus the **steady** amber of a missing/empty slot - so "convert this file" is distinguishable from "nothing here" on the panel. The cause is detected via `IStreamDeck::exists` (present-but-refused = format). (`src/engine/tape/tape_engine.{h,cpp}`)

- **CMake `tape` build was unlinkable (missing `SPK_USE_STREAM`).** The CMake `tape` branch defined only `SPK_ENGINE_TAPE`, but the SD-streaming TUs (`stream_deck`/`fat_file`/`buffer.sdram`) gate their bodies on `SPK_USE_STREAM`, so they compiled empty and the link failed on undefined `StreamDeck`/`FatFile` symbols. The branch now sets `SPK_USE_STREAM`, matching the Makefile. (`CMakeLists.txt`)

## [0.2.4]

### Added

- **Faust-generated reverb engine (`reverb`).** `make ENGINE=reverb` builds a stereo reverb with two switchable algorithms - a Dattorro plate and a Zita-rev1 hall - selected live on Alt+PITCH. The DSP is generated from Faust sources by cyfaust (`make faust-gen`); the kernels are checked in, so the firmware build needs no cyfaust. (`src/engine/reverb/`, `docs/engines/reverb.md`)

- **Tape FX: wow/flutter + tape saturation.** Each tape deck gains an analog-tape effect on its playback signal - wow/flutter into Jiles-Atherton hysteresis/saturation, a per-deck Faust kernel. New per-deck knobs: POS = drive, SIZE = character, MOD_AMT = wow/flutter depth, MODFREQ = rate (POS was previously unused). (`src/engine/tape/`, `docs/engines/tape.md`)

- **gen~ engine support via gen-dsp, and the `gigaverb` engine.** `make ENGINE=gen_gigaverb` builds a stereo reverb authored in Max/MSP **gen~** (Tom Erbe's gigaverb) and translated to C++ by [gen-dsp](https://github.com/shakfu/gen-dsp) - the gen~ counterpart of the Faust path. A host-side generator (`scripts/gen_engine.py`, or `make gen-engines`) turns a gen~ export into a wired-in engine; the generated files are checked in, so the firmware build needs no gen-dsp. gigaverb is 2-in/2-out, 8 params, stereo + knobs only. (`src/engine/gen/`, `src/engine/gen_gigaverb/`, `scripts/gen_engine.py`, `docs/engines/gigaverb.md`)

### Changed

- **Engine docs reorganized into methods vs implementations.** `docs/engines/` now holds only the shipped engines; a new `docs/engine-types/` documents the three ways to build one - native C++, Faust/cyfaust, and gen~/gen-dsp. (`docs/engine-types/`, `docs/engines/`)

- **Edrums now boots silent instead of playing a full kit.** All four drums now seed with zero Euclidean onsets, so the kit boots silent and the player builds the groove up by raising POS on each drum (model, pitch, decay, and length stay pre-seeded, so each drum is voiced the instant its first onset appears). (`src/engine/edrums/edrums_engine.cpp`, `docs/engines/edrums.md`)

## [0.2.3]

### Added

- **Dual streaming tape-deck engine (`tape`).** `make engine-tape` builds two independent **mono** tape decks (A/B) that stream record/playback to/from the SD card, removing the in-SDRAM loop-length cap (~42 s float / ~84 s int16, `kSourceMaxSeconds`) - each deck plays/records an arbitrarily long file. A deck is play-XOR-record (no overdub), so the two run like a pair of record decks (play A while recording B, then play both and beat-match by ear). The engine is thin: in the audio ISR it only moves float frames between the platform's lock-free per-deck SDRAM rings and the buffers; the slow FatFs I/O runs in a dual-deck `StreamDeck` pump on the main loop, reached via a per-deck `IStreamDeck*` injected through `EngineContext`. Native mono float WAV (no conversion); one 1 MB ring per deck + a 32 KB shared scratch; all guarded by `#if defined(SPK_ENGINE_TAPE)` so other engines stay byte-identical. `CapOwnDisplay | CapDualDeck | CapAux | CapAltPos`; NOT `CapTapeStorage`. Dual-deck core hardware-verified; ~81.5% `SRAM_EXEC`. See `docs/engines/tape.md`, `docs/dev/tape-impl.md`. (`src/engine/tape/`, `src/hw/{fat_file,stream_deck}.{h,cpp}`, `src/memory/{spsc_ring,audio_stream,byte_file,wav_stream,wav}.h`, `src/engine/istreamdeck.h`, `src/hw/buffer.sdram.{h,cpp}`, `src/engine/engine_context.h`, `src/app.cpp`, `src/engine/engine_select.h`, `Makefile`, `CMakeLists.txt`, `Makefile.cmake`)

- **Tape transport + per-deck controls.** Per deck: **Play** = play, **Alt+Play** = record (300 ms debounce; Rev pad reserved for reverse); **PITCH** = varispeed (`exp2((v-0.5)*2)`, 0.5x..2x, +/-1 octave); **MIX** = playback volume. Two MONO inputs (deck A records input A, deck B input B - never summed) mix to a stereo bus driving the headphone + the individual jacks. Display per deck: idle ring off / green = play / red = record / amber = rejected start, on the Play-pad LED + ring.

- **Tape routing + pan.** The **routing switch** (`Route`) picks the topology - LEFT = each deck panned by its **Alt+POS** knob, CENTRE = both centered (mono sum), RIGHT = random pan - shown on the mode L/C/R LED. The **mix fader** (`Crossfade`) blends deck A/B (DJ-style: centre = both full, ends = one only).

- **Tape loop modes (ENV knob).** Per deck, by quadrant: none / plain loop / faded loop (~50 ms seam de-click) / Frippertronics (decays ~0.6x per pass, then auto-stops). Looping lives in the stream (`PlayStream` rewinds the WAV at EOF); the fade/decay is engine-side via `loop_frames` + source-frame tracking, with a Frippertronics fade-out flagged from the ISR and stopped in `prepare()`. (`src/memory/{audio_stream,wav_stream}.h`)

- **Tape slots - non-destructive takes.** 8 slots per deck under `/tapes/` (`tape_a_<n>.wav` / `tape_b_<n>.wav`), selected by **Alt+PITCH** (the ring shows the 8-slot selector while Alt is held); the next Play/record targets the active slot. The selector distinguishes **recorded vs empty** slots (selected bright / recorded mid / empty dim), probed via a new `IStreamDeck::exists` (`f_stat`) from the engine's `prepare()` when the selector opens. `FatFile::open_write` creates `/tapes` on first record; single-digit slots keep the names 8.3-safe. (`src/hw/fat_file.cpp`, `src/hw/stream_deck.{h,cpp}`, `src/engine/istreamdeck.h`, `src/engine/tape/tape_engine.{h,cpp}`)

- **`ParamId::AltPos` + `CapAltPos`: a reusable Alt+POS knob layer.** Mirrors `CapAux`/`Aux` (Alt+PITCH): an engine that advertises `CapAltPos` receives the Alt+POS knob as `ParamId::AltPos` (the tape engine reads it as per-deck pan); engines without it keep POS -> `Pos` byte-for-byte, so granular/delay/edrums/passthrough are unchanged. (`src/engine/engine_params.h`, `src/ui/core.ui.{h,cpp}`, `src/engine/granular/granular_engine.cpp`)

- **Headless streaming test.** `make -C host test-stream` builds and runs `host/test_stream.cpp`, an end-to-end exercise of the streaming core over an in-memory `IByteFile` (ISR -> SPSC ring -> streaming-WAV file -> ring -> ISR), asserting a byte-exact round-trip plus the underrun/overrun, EOF, finalize-header-patch, and **loop-rewind** (a looping `PlayStream` repeats the WAV body seamlessly and never finishes) behaviour off-target. The core (`spsc_ring`, `audio_stream`, `wav_stream`, `byte_file`) is engine-agnostic and proven on the host before any device run. (`host/test_stream.cpp`, `host/Makefile`)

### Fixed

- **Tape build keeps the SD card mounted for the session.** `Storage::process()` unmounts the card via `_can_unmount()` once settings are read and both decks are idle (`f_mount(NULL)` + `_fsi.DeInit()`), which tore the FatFs volume out from under the tape stream - every `f_open` then returned `FR_NOT_READY`, so record/play silently no-op'd and no tape file was ever created. Under `#if defined(SPK_ENGINE_TAPE)` the unmount is skipped so the volume stays mounted (the tape engine owns all SD I/O and `Storage` stays idle in that build); non-tape builds are unchanged. Found during hardware bring-up; see `docs/dev/tape-impl.md`. (`src/memory/storage.h`)

## [0.2.2]

### Added

- **Resonator / pluck engine (`reso`).** `make engine-reso` builds a dual resonator/pluck voice on the Mutable Instruments **Rings** modal/string DSP (one `rings::Part` per deck, mono, polyphony 1) rather than a hand-rolled Karplus-Strong. A per-deck **mode switch** sets how the resonator is excited - **Slice** (discrete internal plucks), **Reel** (fed from the live input plus a short ~4 ms trigger burst), **Drift** (a free-running scatter cloud auto-strumming at randomized intervals) - and **Alt+PITCH** (`ParamId::Aux`) selects the Rings model (modal / sympathetic-string / string / FM / string+reverb). Controls: PITCH = note (5-octave C1..C6), SIZE = damping/decay, POS = excitation position, ENV = brightness, MOD_AMT = structure, SOS = dry/wet, MODFREQ = Drift density / Slice arp rate. The V/Oct CV input transposes the note *additively* on top of the knob (0 V = knob only). Advertises `CapOwnDisplay | CapDualDeck | CapAux | CapTransport`; deck A -> left out, deck B -> right out, soft-clipped, with a per-deck energy meter, pitch dot, and an Alt+PITCH model-selector readout shown while Alt is held (the 5 models around the ring, selected one bright) on the rings. Uses PIMPL (an opaque `Impl*`) to keep all Rings/stmlib types out of the header - mandatory because stmlib declares a global `namespace impl` that otherwise collides with `app.cpp`'s composition-root instance. Built at `-Os` (Rings overflows execution SRAM at `-O2`); fits ~96% (183 KB / 186 KB). (`src/engine/reso/`, `src/engine/engine_select.h`, `Makefile`, `CMakeLists.txt`)

- **Vendored Rings/stmlib DSP, trimmed and colocated under the engine.** reso's Rings + stmlib sources live under `src/engine/reso/thirdparty/` (colocated with the only engine that uses them; the `RESO_TP`/`RESO_INC` make vars scope the source list and the `-I` to the reso build, so other engines never see them). The tree is pruned to the exact compile closure - computed from the compiler's own `.d` dependency output - at ~34 files / ~320 KB, down from the ~50 MB upstream checkouts (the bulk was `stmlib/third_party` and Rings' `hardware_design`/`drivers`/`ui`/`bootloader`). stmlib's `LICENSE` is kept; Rings' MIT notice lives in each retained source-file header. (`src/engine/reso/thirdparty/`, `Makefile`, `host/Makefile`)

- **Headless reso test.** `make -C host test-reso` builds and runs `host/test_reso.cpp`, which drives `ResoEngine` through the public `IEngine` surface (stub `ITransport`): param round-trip + mode-switch reporting, Slice decay, Reel quiet-idle / sounds-on-trigger, Drift scatter, MIDI note, autocorrelation pitch-tracking, brightness control, the **V/Oct-CV-must-not-clobber-the-knob** regression, and the **audible-default-ENV** guard (default ENV plucks; ENV fully CCW is near-silent). (`host/test_reso.cpp`, `host/Makefile`)

- **Opt-in CMake build (parity-targeted alternative to the Make build).** A second build system lands alongside the canonical `Makefile`, driven by `make -f Makefile.cmake ENGINE=<engine>` - a thin frontend preserving the `make` command surface (`program-dfu`, `engine-<name>`, `DEBUG=1`, `LOFI_INT16=1`). `CMakeLists.txt` reproduces the Make build's hardware-critical specifics rather than CMake defaults: `BOOT_APP` (skips the SRAM bootloader's re-`SystemInit`), the split opt levels (libs `-O3`, app `-O2`, reso `-Os`), the `alt_sram.lds` linker script, strict `-std=c++17 -fasm -fshort-enums`, and the bleeptools `midi_util.cpp` fork-gap. Each engine caches in its own `build-cmake/<engine>/` dir, so switching engines never forces a rebuild (retiring the `.engine-stamp` clean/rebuild dance). The Make build stays canonical and untouched; adopting CMake (renaming the frontend to `Makefile`) is a deferred option. Verified: the CMake-built reso firmware flashes over DFU and runs identically to the Make build on hardware (the residual ~4 KB `SRAM_EXEC` delta is section layout, not behaviour). (`CMakeLists.txt`, `Makefile.cmake`, `README.md`)

- **Edrums becomes a four-drum engine (two drums per deck).** Each deck now holds two drums in a slot pair; all four sequence and sound at once, but a deck's seven knobs and its ring address only the **focused** drum. ## [0.1.3]

### Fixed

- **Granular slice fade-out at high pitch/speed (V6).** `Vox::_set_decay_start()` assigned the grain decay-start position only when the fade region was shorter than the slice; in the high-increment, small-slice corner (e.g. playhead increment 8.0 with the minimum slice of 1344 samples, where the fade region 1536 exceeds the slice) it left a stale, history-dependent value. ### Fixed (portability)

- **`divider.h` now includes `<cmath>` for `std::round`** (was relying on `<math.h>`, which only places `round` in `std::` on the ARM toolchain, not in standard libc++). Surfaced by the new host harness; no effect on the firmware build. (`src/core/divider.h`)

- **`WavHeader::size` retyped `size_t` -> `uint32_t`.** The RIFF chunk-size field is a 32-bit field per the WAV spec; `size_t` is 4 bytes only on the 32-bit target, so the 44-byte layout (and its `static_assert`) held there but broke on a 64-bit host and would emit a malformed header on any 64-bit build. Layout-identical on-target (verified: the firmware binary is byte-for-byte unchanged in size); `wav.h` also now includes the `<cstdint>`/`<cstddef>`/`<cstring>` it uses rather than relying on its includers. Surfaced by the host harness. (`src/memory/wav.h`)

- **`follower.cpp` now includes `<cmath>` for `std::exp`** (was relying on `<math.h>`; same std-namespace portability issue as `divider.h`). Surfaced by the host harness; no effect on the firmware build. (`src/core/follower.cpp`)
